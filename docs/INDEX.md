# BG Apiary documentation index

This folder contains project documentation collected during the development of BG Apiary.

## Core project documents

- `ROADMAP.md` - sprint plan from cleanup to version 1.0 and beyond.
- `ARCHITECTURE.md` - current technical architecture and target structure.
- `CHANGELOG.md` - release notes and Sprint 1 cleanup notes.
- `SPRINT_1_CLEANUP.md` - what was done during Sprint 1.

## Current product areas

- Apiary management and GPS: files containing `APIARY`, `GPS`, `LOCATION`.
- Hive experience: files containing `HIVE`.
- Weather and nectar: files containing `WEATHER`, `NECTAR`.
- Visual system: files containing `VISUAL`, `DESIGN`, `ASSET`, `ICON`.
- Audits and release checks: files containing `AUDIT`, `TESTY`, `QUALITY`, `RC`.

## Documentation rule going forward

New sprint documentation should use this format:

```text
docs/SPRINT_X_SHORT_NAME.md
```

Examples:

```text
docs/SPRINT_2_UI_BRANDING.md
docs/SPRINT_3_HIVE_MANAGEMENT.md
docs/SPRINT_4_INSPECTIONS.md
```
